# VRChat Lottery Tool — 仕様書

## 1. 概要

VRChat の「Request Invite」通知を WebSocket で受信し、重み付き抽選を行って当選者に Invite を自動送信する WPF デスクトップアプリケーション。

## 3. 機能仕様

### 3.1 認証

#### ログインフロー

```
1. GET /api/1/auth/user (Basic 認証)
   → requiresTwoFactorAuth が空配列 → ログイン完了
   → ["totp"] → TOTP 入力画面へ
   → ["emailOtp"] → Email OTP 入力画面へ

2. POST /api/1/auth/twofactorauth/totp/verify  (TOTP の場合)
   POST /api/1/auth/twofactorauth/emailotp/verify (Email OTP の場合)
   → auth Cookie を取得

3. Cookie を DPAPI 暗号化して %AppData%\VRChatLotteryTool\.authcookie に保存
```

#### セッション復元

```
起動時:
1. .authcookie から Cookie を読み込む
2. GET /api/1/auth/user で Cookie 有効性を確認
3. 有効 → 自動ログイン完了
4. 無効 → ログインフォームを表示
```

#### 認証情報の保存

- パスワード：Windows DPAPI（ProtectedData.Protect）で暗号化して `.credentials` に保存
- Cookie：`.authcookie` に保存（DPAPI暗号化なし、Cookie値そのまま）
- 有効スコープ：`DataProtectionScope.CurrentUser`（同一ユーザー・同一マシンでのみ復号可能）

### 3.2 Request Invite 受信

#### WebSocket Pipeline 接続

```
接続先: wss://pipeline.vrchat.cloud/?authToken={auth Cookie値}
再接続: 指数バックオフ（初回 5 秒、最大 60 秒）
```

#### メッセージ処理

```json
受信メッセージ形式:
{
  "type": "notification",
  "content": "{\"type\":\"requestInvite\",\"id\":\"...\",\"senderUserId\":\"...\",\"senderUsername\":\"...\"}"
}
```

- `type == "notification"` かつ `content.type == "requestInvite"` のみ処理
- 抽出した `senderUserId`・`senderUsername`・通知 `id` を `RequestInviteEventArgs` に格納してイベント発火

### 3.3 受付管理

#### スケジューラー

`SchedulerService` が Timer を使い以下のイベントを発火：

| イベント | タイミング |
|---|---|
| `ReceptionStarted` | 受付開始時刻 |
| `ReceptionEnded` | 受付終了時刻 |
| `ReplyTimeReached` | 返信時刻 |

#### 受付判定

| 状態 | 処理 |
|---|---|
| `BeforeReception`（受付前） | 応募を記録するが `IsAccepted=false`・理由「受付開始前」 |
| `Accepting`（受付中） | 応募を受理 |
| `ReceptionClosed`（受付終了後） | 応募を記録するが `IsAccepted=false`・理由「受付終了後」 |

#### 重複チェック

同一セッション × 同一 `UserId` の 2 回目以降の Request Invite は：
- 応募数にカウントしない
- `NotificationId` を最新の通知 ID に更新（古い通知の失効対策）

### 3.4 抽選

#### 重み付き抽選アルゴリズム

```
1. 受理済み応募者リストを取得
2. 応募者数 ≤ 当選人数 → 全員当選
3. 応募者数 > 当選人数 → 以下を当選人数回繰り返す：
   a. 残候補全員の重みを計算
   b. 重みの合計に対しランダム値でルーレット選択
   c. 当選者を候補から除外
4. ユーザー統計を更新（TotalWins / LoseStreak / LastWinAt）
```

#### 重み計算

**公平モード**

```
重み = 100
     - [直近24h以内当選] × 15
     - min(累計当選数 × 2, 30)
最小値 = 0.1
```

**救済モード**

```
重み = 100
     + min(連続落選数 × 10, 100)
     + [一度も当選なし] × 30
     - [直近24h以内当選] × 25
     - min(累計当選数 × 5, 50)
最小値 = 0.1
```

### 3.5 Invite 送信

#### APIエンドポイント

```
POST /api/1/invite/{senderUserId}
Body: { "instanceId": "wrld_xxx:12345~hidden(...)~region(jp)~nonce(...)" }
```

#### instanceId 取得フロー

```
1. GET /api/1/users/{自分のUserId} でレスポンスボディから location を取得
2. location の特殊値チェック:
   offline / private / traveling / traveling:traveling → エラー（送信不可）
3. location をそのまま instanceId として使用
   ※ location = "wrld_xxx:instanceId" の形式全体を渡す
```

### 3.6 設定の永続化

設定変更時に即座に `appsettings.json` へ保存：

```json
{
  "WinnerCount": 1,
  "Mode": "Fair",
  "ReceptionStartTime": "21:00",
  "ReceptionEndTime": "22:00",
  "ReplyTime": "22:05"
}
```

## 4. データモデル

### User

```
UserId        : string  (PK)
DisplayName   : string
TotalEntries  : int     (累計応募数)
TotalWins     : int     (累計当選数)
LoseStreak    : int     (現在の連続落選数)
LastWinAt     : datetime?
LastEntryAt   : datetime?
CreatedAt     : datetime
UpdatedAt     : datetime
```

### LotterySession

```
SessionId       : string  (PK)
ReceptionStartAt: datetime
ReceptionEndAt  : datetime
ReplyAt         : datetime
WinnerCount     : int
Mode            : enum (Fair / Relief)
Status          : enum (BeforeReception / Accepting / ReceptionClosed / Drawn / Completed / Error)
DrawExecutedAt  : datetime?
ReplyExecutedAt : datetime?
```

### LotteryEntry

```
EntryId        : string  (PK)
SessionId      : string  (FK → LotterySession)
UserId         : string
DisplayName    : string
NotificationId : string?  (Request Invite 通知ID。重複時は最新に更新)
RequestedAt    : datetime
IsAccepted     : bool
IsWinner       : bool
AppliedWeight  : double?
InviteSent     : bool
InviteSentAt   : datetime?
RejectReason   : string?
ErrorMessage   : string?
```

**インデックス**：`(SessionId, UserId)` にユニーク制約

## 7. VRChat API エンドポイント一覧

| メソッド | エンドポイント | 用途 |
|---|---|---|
| GET | `/auth/user` | ログイン・セッション確認 |
| POST | `/auth/twofactorauth/totp/verify` | TOTP 認証 |
| POST | `/auth/twofactorauth/emailotp/verify` | Email OTP 認証 |
| PUT | `/logout` | ログアウト |
| GET | `/users/{userId}` | ユーザー情報・location 取得 |
| GET | `/worlds/{worldId}` | ワールド名取得 |
| POST | `/invite/{userId}` | Invite 送信（Request Invite 承認） |
| WebSocket | `wss://pipeline.vrchat.cloud/` | Request Invite 受信 |

## 8. 既知の制限

- Invite 送信は自分が VRChat のワールドに入室していないと失敗する
- テスト送信（Simulate）で発生した疑似応募は `NotificationId` が空のため実際の Invite は送信されない
- WebSocket Pipeline は接続が切れると自動再接続するが、その間の通知は受信できない
